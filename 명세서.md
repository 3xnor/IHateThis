# 유스케이스 명세서 — 한국어 스팸 메일 분류 시스템

> 작성자: Systems Analyst
> 기준 문서: UCS v1.1
> 작성일: 2026-03-23

---

| UC ID | 유스케이스명 | 사전 조건 | 사후 조건 | 기본 흐름 |
|-------|------------|----------|----------|---------|
| UC-01 | 회원가입 | 사용자가 서비스에 가입되어 있지 않다 | 계정이 생성되고 로그인 가능한 상태가 된다 | 1. 사용자가 이메일·비밀번호·비밀번호 확인 입력<br>2. 시스템이 이메일 형식 및 비밀번호 규칙 검증<br>3. 시스템이 이메일 중복 여부 확인<br>4. 비밀번호 bcrypt 해시 암호화 후 DB 저장<br>5. 회원가입 완료 메시지 반환 |
| UC-02 | 로그인 | 사용자가 회원가입을 완료한 상태이다 | Access Token(15분)·Refresh Token(7일)이 발급되어 인증 API 호출이 가능한 상태가 된다 | 1. 사용자가 이메일·비밀번호 입력 후 `POST /auth/login` 호출<br>2. 시스템이 DB에서 이메일로 사용자 조회<br>3. 입력 비밀번호와 해시 비교<br>4. Access Token·Refresh Token 생성 및 반환<br>5. 클라이언트가 이후 요청에 `Authorization: Bearer <token>` 포함 |
| UC-04 | 단일 이메일 스팸 분류 | 사용자가 로그인 상태이며 사용할 모델(ml 또는 bert)이 로드된 상태이다 | 분류 결과가 반환되고 이력 DB에 저장된다 | 1. 사용자가 `POST /predict`에 subject·body·model 전송<br>2. Preprocessor로 텍스트 정제 (HTML 제거 → 토큰화 → 특수문자 정리 → 공백 정규화)<br>3. 선택 모델로 예측 수행<br>4. 스팸 확률 ≥ 0.7이면 spam, 미만이면 ham 판정<br>5. label·confidence·spam_probability·ham_probability·processing_time_ms 반환<br>6. 요청·결과·사용자 ID를 이력 DB에 저장 |
| UC-05 | 대량 이메일 일괄 분류 | 사용자가 로그인 상태이며 사용할 모델이 로드된 상태이다 | 각 이메일의 분류 결과 목록과 전체 처리 시간이 반환된다 | 1. 사용자가 `POST /predict/batch`에 이메일 목록(최대 100건) 전송<br>2. 시스템이 동일 모델 요청끼리 묶어 배치 추론 수행<br>3. 각 이메일에 대해 UC-04의 3~5단계 수행<br>4. 전체 결과 목록과 total_processing_time_ms 반환 |
| UC-06 | Gmail 계정 연동 및 스팸 분류 | 사용자가 로그인 상태이며 사용할 모델이 로드된 상태이다. Google Cloud Console에서 Gmail API가 활성화되고 OAuth 2.0 클라이언트가 등록된 상태이다 | 가져온 메일 각각의 스팸 분류 결과가 반환된다. 사용자의 OAuth 토큰이 서버에 암호화 저장된다 | **[연동]**<br>1. 사용자가 `GET /gmail/connect` 호출<br>2. 시스템이 Google OAuth 2.0 인증 URL 생성·반환 (scope: gmail.readonly)<br>3. 사용자가 브라우저에서 Google 권한 동의 완료<br>4. Google이 인증 코드를 `GET /gmail/callback`으로 전달<br>5. 시스템이 인증 코드를 Access Token·Refresh Token으로 교환<br>6. 시스템이 OAuth 토큰을 사용자 DB에 암호화 저장<br>7. Gmail 연동 완료 메시지 반환<br>**[분류]**<br>8. 사용자가 `POST /gmail/classify`에 max_results·label_filter·model 전송<br>9. 시스템이 OAuth 토큰으로 Gmail API 호출하여 메일 목록 수신<br>10. 각 메일의 subject·body 파싱 (base64 디코딩, multipart 처리, HTML 제거)<br>11. UC-05 방식으로 스팸 분류 수행<br>12. gmail_id·subject·sender·date·label·confidence·spam_probability 반환<br>**[해제]**<br>13. 사용자가 `DELETE /gmail/disconnect` 호출 시 OAuth 토큰 삭제 및 Google 토큰 폐기 |
| UC-07 | 데이터셋 조회 | 관리자 권한으로 로그인한 상태이다 | 데이터셋 통계와 샘플 데이터가 반환된다 | 1. 관리자가 `GET /admin/dataset` 호출<br>2. 시스템이 korean_spam_dataset.csv 로드 및 통계 집계 (전체·스팸·정상 건수, 카테고리별 분포, 평균 제목·본문 길이)<br>3. 통계와 상위 5건 샘플 데이터 반환 |
| UC-09 | ML 모델 학습 | 관리자 권한으로 로그인한 상태이며 korean_spam_dataset.csv가 존재한다 | artifacts/ml_model.joblib이 저장되고 성능 지표가 기록된다 | 1. 관리자가 `POST /admin/train/ml`에 max_features·ngram_range·C·spam_threshold 전송<br>2. CSV 로드 및 전처리 수행<br>3. 데이터 train(80%) / val(10%) / test(10%) 분리<br>4. MLSpamClassifier 학습 (class_weight='balanced')<br>5. Validation 세트로 중간 결과 출력<br>6. Test 세트로 최종 Precision·Recall·F1 계산<br>7. 모델을 artifacts/ml_model.joblib으로 저장<br>8. 성능 지표를 artifacts/ml_metrics.json으로 저장<br>9. 학습 완료 메시지 및 성능 요약 반환 |
| UC-10 | DL(BERT) 모델 학습 | 관리자 권한으로 로그인한 상태이며 korean_spam_dataset.csv가 존재한다. CUDA GPU 사용 시 드라이버 설치 권장 | artifacts/bert_model/에 모델 파일이 저장되고 성능 지표가 기록된다 | 1. 관리자가 `POST /admin/train/bert`에 base_model·max_length·batch_size·epochs·learning_rate·warmup_ratio·weight_decay·spam_threshold 전송<br>2. CSV 로드 및 전처리 수행<br>3. 데이터 train(80%) / val(10%) / test(10%) 분리<br>4. HuggingFace Hub에서 KR-ELECTRA 모델·토크나이저 다운로드<br>5. 에폭마다 mini-batch 학습 (AdamW + Linear Warmup 스케줄러), validation F1 계산, 최고 F1 체크포인트 저장 (Early Stopping)<br>6. Best 체크포인트를 artifacts/bert_model/에 저장<br>7. Test 세트로 최종 Precision·Recall·F1 계산<br>8. 성능 지표를 artifacts/bert_metrics.json으로 저장<br>9. 학습 완료 메시지 및 에폭별 학습 곡선 데이터 반환 |
