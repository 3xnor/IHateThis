UC-01 회원가입 일반 사용자
사전 조건
PRE
사용자가 서비스에 가입되어 있지 않다
사후 조건
POST
계정이 생성되고 로그인 가능한 상태가 된다
기본 흐름
  1. 1사용자가 이메일·비밀번호·비밀번호 확인 입력
  2. 2시스템이 이메일 형식 및 비밀번호 규칙 검증 (8자 이상, 영문+숫자)
  3. 3시스템이 이메일 중복 여부 확인
  4. 4비밀번호를 bcrypt 해시로 암호화 후 DB 저장
  5. 5회원가입 완료 메시지 반환
대안 흐름 & 예외 흐름 대안조건 불충족 시 정상 종료 불가 예외시스템 오류
ID유형분기 단계발생 조건처리 결과
1a 대안 2단계 분기 이메일 형식이 RFC 5322 규격을 위반한 경우 (예: @ 없음, 도메인 없음) 400 "이메일 형식이 올바르지 않습니다." 반환 후 흐름 종료
1b 대안 2단계 분기 비밀번호가 8자 미만이거나 영문·숫자 중 하나라도 미포함인 경우 400 "비밀번호는 8자 이상이며 영문과 숫자를 포함해야 합니다." 반환 후 흐름 종료
1c 대안 2단계 분기 비밀번호 확인 값이 비밀번호와 일치하지 않는 경우 400 "비밀번호가 일치하지 않습니다." 반환 후 흐름 종료
2a 대안 3단계 분기 입력 이메일이 이미 DB에 존재하는 경우 409 "이미 사용 중인 이메일입니다." 반환 후 흐름 종료
3a 예외 4단계 분기 DB 저장 중 연결 오류 또는 트랜잭션 실패 발생 500 "서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요." 반환, 저장 롤백

3a 예외 4단계 분기 JWT 토큰 서명 키 누락 또는 토큰 생성 중 시스템 오류 발생 500 "토큰 발급에 실패했습니다." 반환
UC-04 단일 이메일 스팸 분류 일반 사용자
사전 조건
PRE
사용자가 로그인 상태이며, 사용할 모델(ml 또는 bert)이 로드된 상태이다
사후 조건
POST
분류 결과가 반환되고 이력 DB에 저장된다
기본 흐름
  1. 1POST /predict에 subject·body·model 전송
  2. 2Preprocessor로 텍스트 정제 (HTML 제거 → 토큰화 → 특수문자 정리 → 공백 정규화)
  3. 3선택 모델로 예측 수행
  4. 4스팸 확률 ≥ 0.7 이면 spam, 미만이면 ham 판정
  5. 5label·confidence·spam_probability·ham_probability·processing_time_ms 반환
  6. 6요청·결과·사용자 ID를 이력 DB에 저장
대안 흐름 & 예외 흐름 대안 예외
ID유형분기 단계발생 조건처리 결과
1a 대안 1단계 분기 요청 헤더에 Authorization 토큰이 없거나 만료된 경우 401 "인증이 필요합니다." 반환, 이하 흐름 미실행
1b 대안 1단계 분기 subject 또는 body 필드가 빈 문자열이거나 누락된 경우 422 "제목과 본문을 입력해주세요." 반환
1c 대안 1단계 분기 model 파라미터가 "ml" 또는 "bert" 이외의 값인 경우 422 "model은 'ml' 또는 'bert'만 허용됩니다." 반환
2a 예외 3단계 분기 요청한 모델이 서버에 로드되지 않은 상태인 경우 503 "해당 모델이 로드되지 않았습니다. 먼저 학습을 실행하세요." 반환
3a 예외 6단계 분기 분류는 완료됐으나 이력 DB 저장 중 오류 발생 분류 결과는 정상 반환, 이력 저장 실패는 서버 로그에만 기록 (사용자에게 별도 오류 미노출)

UC-06 Gmail 계정 연동 및 스팸 분류 일반 사용자 Gmail 서버
사전 조건
PRE
사용자가 로그인 상태이며 모델이 로드된 상태이다
PRE
Google Cloud Console에서 Gmail API 활성화 및 OAuth 2.0 클라이언트 등록 완료
사후 조건
POST
가져온 메일 각각의 스팸 분류 결과가 반환된다
POST
사용자의 OAuth 토큰이 서버에 암호화 저장된다
기본 흐름
연동
  1. 1GET /gmail/connect 호출
  2. 2시스템이 Google OAuth 2.0 인증 URL 생성·반환 (scope: gmail.readonly)
  3. 3사용자가 브라우저에서 Google 권한 동의 완료
  4. 4Google이 인증 코드를 GET /gmail/callback으로 전달
  5. 5시스템이 인증 코드를 Access Token·Refresh Token으로 교환
  6. 6OAuth 토큰을 사용자 DB에 암호화 저장
  7. 7Gmail 연동 완료 메시지 반환
분류
  1. 8POST /gmail/classify에 max_results·label_filter·model 전송
  2. 9OAuth 토큰으로 Gmail API 호출, 메일 목록 수신
  3. 10각 메일 파싱 (base64 디코딩, multipart 처리, HTML 제거)
  4. 11UC-05 방식으로 스팸 분류 수행
  5. 12gmail_id·subject·sender·date·label·confidence·spam_probability 반환
해제
  1. 13DELETE /gmail/disconnect 호출 시 OAuth 토큰 삭제 및 Google 토큰 폐기
대안 흐름 & 예외 흐름 대안 예외
ID유형분기 단계발생 조건처리 결과
1a 대안 3단계 분기 사용자가 Google 권한 동의 화면에서 "취소"를 선택한 경우 403 "Gmail 접근 권한이 거부되었습니다." 반환, 토큰 저장 미실행
2a 대안 5단계 분기 콜백으로 전달된 인증 코드가 발급 후 10분 이상 경과하여 만료된 경우 400 "인증 코드가 만료되었습니다. 다시 시도해주세요." 반환, 1단계부터 재시작 안내
3a 대안 9단계 분기 저장된 Gmail Access Token이 만료된 경우 (통상 1시간) 자동 처리 Refresh Token으로 새 Access Token 재발급 후 Gmail API 재호출, 사용자에게 노출 없음
3b 대안 9단계 분기 Refresh Token도 만료되거나 사용자가 Google에서 앱 권한을 직접 해제한 경우 401 "Gmail 재연동이 필요합니다." 반환, DB의 OAuth 토큰 삭제 처리
4a 예외 9단계 분기 Gmail API 서버 응답 오류 (5xx) 또는 네트워크 타임아웃 발생 502 "Gmail 서버와 통신에 실패했습니다." 반환, 1회 자동 재시도 후에도 실패 시 오류 반환
4b 예외 10단계 분기 특정 메일의 MIME 구조가 비표준이거나 base64 디코딩에 실패한 경우 해당 메일의 body를 빈 문자열로 대체하여 분류 계속 진행, 응답에 parse_error: true 플래그 포함

UC-10 DL(BERT) 모델 학습 관리자
사전 조건
PRE

PRE

사후 조건
POST
artifacts/bert_model/ 저장, 성능 지표 기록
기본 흐름
  1. 1POST /admin/train/bert에 학습 파라미터 전송
  2. 2CSV 로드 및 전처리 수행

  6. 6Best 체크포인트를 artifacts/bert_model/에 저장
  7. 7Test 세트로 최종 Precision·Recall·F1 계산
  8. 8성능 지표를 artifacts/bert_metrics.json으로 저장
  9. 9학습 완료 메시지 및 에폭별 학습 곡선 데이터 반환

1b 대안 1단계 분기 이미 BERT 또는 ML 학습이 진행 중인 경우 (중복 요청) 409 "이미 학습이 진행 중입니다. 완료 후 재시도하세요." 반환
2a 예외 4단계 분기 HuggingFace Hub 네트워크 오류 또는 모델명이 존재하지 않는 경우 502 "베이스 모델 다운로드에 실패했습니다. 모델명 및 네트워크를 확인하세요." 반환
3a 예외 5단계 분기 GPU VRAM 부족(CUDA OOM) 오류 발생 자동 재시도 batch_size를 절반으로 줄여 1회 재시도. 재시도 후에도 OOM 발생 시 500 "GPU 메모리가 부족합니다. batch_size를 줄이거나 CPU 학습을 사용하세요." 반환
3b 예외 5단계 분기 학습 중 NaN loss 발생 또는 gradient 폭발로 학습 불가 상태 500 "학습이 불안정합니다. learning_rate를 낮추거나 warmup_ratio를 높여 재시도하세요." 반환, 부분 체크포인트 삭제
4a 예외 6단계 분기 Early Stopping 발동 없이 전체 에폭 완료 시 best 체크포인트가 없는 경우 (첫 에폭부터 F1 하락) 마지막 에폭의 가중치를 best로 간주하여 저장 후 경고 메시지 포함 반환: "성능 개선이 감지되지 않았습니다. 하이퍼파라미터 조정을 권장합니다."
